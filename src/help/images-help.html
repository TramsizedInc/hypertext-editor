Images, Media and Embedding

Why are my images broken?

If you try to enter a link to an image, media and other file stored locally on your computer relative to your document, it won't be viewable while editing because of browser engine security protections. Though if you save the document and then open the file in your web browser, it will work as expected (if the relative URL is entered correctly.)

Solutions: 

1. The first solution is to include images from websites using the absolute URL like: https://example.com/test.png .

2. The second option is that you can embed the image inside the document itself by clicking the "upload" button:  in the images dialog box next to the "Source" input box. (That's how the previous image was included in this document). The editor will convert the image into textual data that can be stored directly in the page. This will make the markup of the page messy and the file size larger, but has the benefit of always being viewable. 

3. You can start a small web server on your desktop and serve the folder where your HTML document will be saved. To use this server, open the Tools â†’ Preferences dialog box and enter the local web server address (see below) into the "Local Server" input field. The document will then use that server as the base address for any relative URLs such as images or media.

The simplest way to run a local web server is to use Python. If you are on Windows, you will need to install it from https://www.python.org/downloads/. Mac users will have it pre-installed, and Linux users will figure it out, I'm sure.

Open the Command Prompt / Terminal app, and change to your documents directory (e.g. cd Documents). Then run the following: 

python3 -m http.server

Your local server will start serving your documents at http://localhost:8000/
